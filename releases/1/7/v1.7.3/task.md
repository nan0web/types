# 🚀 Mission: Fix TFunction and Parser Error Context

## 🏁 Overview (Огляд)
Реліз v1.7.3 вирішує проблеми з типізацією `TFunction` (другий аргумент має бути опціональним), додає контекст рядків (lineNum, context) у помилки парсера для легшого дебагу `NaN0` формату, а також реалізує базовий fallback для функції перекладу `t` у класі `Model` (підтримка заміни змінних `{key}`).

## 👥 User Stories (Сценарії)
> Як розробник, я хочу викликати `t('key')` без передачі другого аргументу без помилок TypeScript.
> Як розробник, коли парсер падає через неправильний відступ, я хочу бачити номер рядка та його вміст для швидкого фіксу.
> Як користувач базової `Model`, я хочу, щоб дефолтна функція `t` вміла замінювати `{key}` на значення без підключення повноцінного пакету `i18n`.

## 🎯 Scope (Задачі)
- [x] Зробити аргумент `arg1` у `TFunction` опціональним.
- [x] Оновити `Parser.js` та `scanLines`, щоб вони кидали детальні помилки з `lineNum` та контекстом попередніх рядків.
- [x] Оновити базову `Model` для підтримки `{key}`-шаблонізації в дефолтній функції `t`.

## ✅ Acceptance Criteria (DoD)
- [ ] Контрактні тести (`task.spec.js`) написані і успішно проходять (Green).
- [ ] `TFunction` не вимагає другого аргументу в `.d.ts`.
- [ ] Оновлено CHANGELOG.md.
