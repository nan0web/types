# Інтернаціоналізація (i18n)

Пакет `@nan0web/types` надає легкий механізм i18n, спроектований для одночасної простоти та архітектурної надійності у високонавантажених JS-додатках.

## `createT(vocabulary, locale)`

Створює функцію перекладу, яка підтримує підстановку змінних, рекурсивну множину та скорочений запис для чисел.

```js
import { createT } from "@nan0web/types"
const t = createT({
    'Hello {name}': 'Привіт, {name}!',
    apples_one: '{count} яблуко',
    'I have {apples}': 'У мене є {apples}',
}, 'uk-UA')

console.info(t('Hello {name}', { name: 'Світ' })) // "Привіт, Світ!"
console.info(t('I have {apples}', { apples: 5 })) // "У мене є 5 яблук"
```

### `TFunction` (Контракт)
Визначення типу функції для перекладів:
`(key: string, vars?: Record<string, any>) => string`

---

## 🏗 Симетрична валідація i18n

Цей пакет впроваджує патерн **Симетричної валідації i18n**.

### Проблема
Традиційна валідація повертає статичні рядки, які важко перекладати на рівні інтерфейсу (CLI, Web, Mobile).

### Рішення: "Помилки з домену" (Domain-First Errors)
Замість того, щоб повертати статичні рядки з правил валідації, ваші доменні моделі повертають **структури для перекладу** (наприклад, `["Invalid DSN: {dsn}", { dsn: value }]`).

Ці помилки:
1.  **Визначаються** в доменних моделях за допомогою простих масивів/об'єктів.
2.  **Перехоплюються** додатком як `ModelError`.
3.  **Перекладаються** на рівні інтерфейсу за допомогою локального інстансу `TFunction`.

### `ModelError`
Структурований клас помилок для збоїв валідації. Він містить мапу полів, що не пройшли валідацію, та відповідні повідомлення або конфігурації перекладу, зберігаючи контекст (префікс $) для множини та підстановок.

```js
import { resolveValidation, ModelError } from "@nan0web/types"
class User {
    static name = { 
      errorTooShort: 'Name too short', // важливо використовувати всі переклади як static error*, label*, help*, placeholder*, детальніше у @nan0web/i18n
      validate: (v) => v.length > 2 || User.name.errorTooShort
    }
}

try {
    resolveValidation(User, { name: 'Bo' })
} catch (e) {
    if (e instanceof ModelError) {
        console.info(e.fields) // { name: "Name too short" }
    }
}
```
